# Wildcard Mask in Cisco IOS

A wildcard mask is a bit pattern used in Cisco IOS to define which parts of an IP address should be examined for matching purposes. Unlike subnet masks, wildcard masks invert the bits, where:

- `0` indicates the bit must match exactly
- `1` indicates the bit can be any value

## Key Uses
Wildcard masks are used in:
- Defining network ranges in routing protocols (e.g., OSPF)
- Configuring access control lists (ACLs) to permit/deny IP addresses

## How Wildcard Masks Work
Example:  
Subnet mask `255.255.255.0` (binary: `11111111.11111111.11111111.00000000`)  
Wildcard mask: `0.0.0.255` (binary: `00000000.00000000.00000000.11111111`)

## Matching Examples
- Wildcard `0.0.0.254` (binary: `11111110`) matches:
  - Even IPs: `10.10.10.0`, `10.10.10.2`, `10.10.10.4`...
  - Odd IPs: `10.10.10.1`, `10.10.10.3`, `10.10.10.5`...
